# Bitwig Studio 実演デモ手順書

この資料は、サウンドデザイン講義の「音の操作」デモを Bitwig Studio を用いて実演する場合の手順と設定方法をまとめたものです。
Bitwig の強力なモジュレーションシステム（Modulators）を活用することで、Max と同等の「視覚的な結線（マッピング）」を直感的に提示できます。

---

## 基本的なセットアップ（講義前）

1. **プロジェクトの準備**
   - デモごとに別々の **Instrument Track** を作成しておく。
   - 各トラックに **Poly Grid** または **Sampler** をインサートする。
   - 画面下部の **Device Panel** を開き、モジュレーター（矢印のアイコン）を展開して見せやすい状態にしておく。

2. **視覚化の準備**
   - マスターの出力、または各トラックの最後に **Oscilloscope**（波形表示）や **Spectrum**（周波数表示）デバイスを入れておく。これにより、音の物理的変化を視覚で証明できます。

---

## 1. エンベロープで操作する（時間の形をデザインする）
対象デバイス：**Poly Grid**（サイン波/ノコギリ波）または **Sampler**

### デモ1.1：音量（アタック感と余韻）
- **対象**：デバイス内の `Volume` または `Amp`
- **モジュレーター**：`ADSR`
- **手順**：
  1. `ADSR` モジュレーターを追加し、`Amp` パラメータにマッピングする。
  2. ノートを入力（MIDIキーボードまたはクリップ）して音を鳴らす。
  3. `A (Attack)` をゼロにして「クリック・接触感」を見せ、次に `A` を遅くして「配慮のある通知」を見せる。
  4. `R (Release)` をバッサリ切って「エラー感（余韻の強制終了）」を見せる。

### デモ1.2：ピッチ（エネルギーの移動）
- **対象**：オシレーター/サンプラーの `Pitch` パラメータ
- **モジュレーター**：`ADSR` または `AHDSR`
- **手順**：
  1. `ADSR` を追加し、`Pitch` にマッピングする（変調量は +24st や -24st など大きめに設定）。
  2. `Decay` や `Release` を短く設定し、音が急降下する「レーザー音」や「動物の咆哮（サンプラー使用時）」を実演する。
  3. 変調量をプラスにして「ジャンプ音（上昇）」を見せる。

### デモ1.3：フィルター（質感の変化）
- **対象**：デバイス内の `Filter`（Low-pass）の `Cutoff Frequency`
- **モジュレーター**：`ADSR` または `Envelope`
- **手順**：
  1. オシレーターを倍音の多い波形（Sawtoothなど）、または声のサンプルにする。
  2. フィルターの Cutoff を低く（こもった音に）設定しておく。
  3. `ADSR` を Cutoff にマッピングし、ノート入力時にフィルターが開くようにする。
  4. アタックとディケイを調整して、「息を吹き込むような有機的なブラス音」を作る過程を見せる。

## 2. LFOで操作する（周期と揺らぎをデザインする）
対象デバイス：**Poly Grid** または **Sampler**

### デモ2.1：ピッチ（ビブラートからアラームへ）
- **対象**：`Pitch` パラメータ
- **モジュレーター**：`LFO`（波形：Sine または Triangle）
- **手順**：
  1. `LFO` を追加し、`Pitch` にマッピングする。
  2. `Rate` を遅く、変調量を小さく設定し、感情的な「ビブラート」を見せる。
  3. 徐々に `Rate` を速く（例：5Hz〜10Hz）、変調量を大きく（数半音）し、「サイレン・緊急アラーム」へと意味が変化する様を見せる。

### デモ2.2：音量（トレモロと注意喚起）
- **対象**：`Volume` または `Amp`
- **モジュレーター**：`LFO`（波形：Square または Pulse）
- **手順**：
  1. アンビエントパッドや環境音（カフェのガヤガヤなど）をループ再生する。
  2. `LFO`（波形を四角いパルス波に設定）を `Amp` にマッピングし、音を周期的にON/OFFさせる。
  3. 「背景音の中に埋もれない強制的な通知」として、聴覚特性（変化に敏感）を利用していることを解説する。

---

## 3. インタラクション・システム状況で操作する
対象デバイス：**Sampler** および **Note Grid**

### デモ3.1：再生ヘッドの位置 × マウス入力（手触りのデザイン）
- **対象**：**Sampler** の `Playhead` (再生位置)
- **モジュレーター**：`Macro` または `Button`（手動操作用）
- **手順**：
  1. 声などの分かりやすいオーディオを Sampler に入れる。
  2. Sampler の再生モードを `Texture` または `Cycles`（グラニュラーモード）に設定。
  3. `Macro` ノブを追加し、Sampler の再生開始位置（Start）にマッピングする。
  4. `Macro` ノブをマウスで左右にグリグリ動かし、音がタイムストレッチされたり逆再生される「デジタルデータの物理的な手触り」を実演する。

### デモ3.2：発音確率 × システムのエラー率（不安定さの表現）
- **対象**：MIDIノートの発生確率
- **デバイス構成**：**Note Grid** → **Sampler**（足音やクリック音）
- **手順**：
  1. Note Grid 内で `Transport`（Metro）→ `Chance`（確率）→ `Note Out` のパッチを組む。
  2. `Chance` ノジュールの `Probability` パラメータ（0〜100%）に `Macro` ノブをマッピングする。
  3. Macro ノブが100%の時は「規則的で確実な足音」が鳴るのを見せる。
  4. システムの疲労やエラーを模して Macro ノブを徐々に下げていく。音が不規則に欠落し、「不安定さ（バグ感）」が生じる様子を実演する。

## Bitwig で実演する際のポイント

- **視覚的なフィードバックの強調**
  Bitwig の最大の特徴は、モジュレーターが対象のパラメータノブを「実際に揺らしている（リングが動く）」様子が視覚的に見えることです。デモの際は、このアニメーションをプロジェクターでしっかり見せ、「見えない力を結びつけている（マッピング）」ことを強調してください。

- **ドラッグ＆ドロップの活用**
  Sampler デバイスの波形表示画面に、OSのフォルダからオーディオファイルを直接ドラッグ＆ドロップする様子を見せます。「オシレーターからリアルな録音データへ」という Phase 1 → Phase 2 の移行が、ドラッグ＆ドロップ1回でシームレスに行えることをアピールします。
