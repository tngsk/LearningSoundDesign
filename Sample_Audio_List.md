# サウンドデザイン講義 デモ用サンプル音源リスト

このリストは、Maxパッチ（demo_1_1 〜 demo_3_2）での実演効果を最大化するために準備しておくべきオーディオファイルのリストです。
すべてのファイルは `16bit / 44.1kHz の WAV形式` で準備し、講義中は一つのフォルダにまとめておき、Maxの `dropfile` UIへすぐにドラッグ＆ドロップできるようにしておいてください。

---

## 1. エンベロープ用（時間の形をデザインする）
音の「立ち上がり（アタック）」と「余韻（リリース）」を切り取り、物理的な重さやエネルギー感の変容を見せます。

### 必要なファイル
- [ ] `white_noise.wav` または `tv_static.wav` (長めの砂嵐ノイズ)
  - **用途**: [demo_1_1_env_amp] アタックを短く切り、ノイズをUIの「クリック音」に変換する。
- [ ] `bell_ring_long.wav` (余韻の長いベルやシンギングボウル)
  - **用途**: [demo_1_1_env_amp] リリースを極端に短くし、余韻を強制終了させることで「エラー感」を演出する。
- [ ] `lion_roar.wav` または `dog_bark.wav` (動物の咆哮)
  - **用途**: [demo_1_2_env_pitch] ピッチエンベロープで一瞬で音程を急降下させ、SFの巨大なモンスターやビーム兵器の音を作る。
- [ ] `human_breath.wav` (息を強く吹き込む音)
  - **用途**: [demo_1_3_env_filter] フィルターエンベロープで息の強さに合わせて高域を開き、有機的なシンセブラス音を作る。

---

## 2. LFO用（周期と揺らぎをデザインする）
周期的な変調によって、「ただの持続音」が「警告」や「意味のある情報」に変わる瞬間を見せます。

### 必要なファイル
- [ ] `sine_1kHz.wav` または `telephone_dial.wav` (単調なピーという持続音)
  - **用途**: [demo_2_1_lfo_pitch] LFOでピッチを揺らし、最初は遅く（ビブラート）、徐々に速く深くして「緊急アラーム」へと意味を変容させる。
- [ ] `drone_pad.wav` (アンビエントなシンセパッドや重低音)
  - **用途**: [demo_2_2_lfo_amp] LFOで音量を揺らし（トレモロ）、揺れのスピードを変えることで「緊張感」や「心臓の鼓動」を演出する。
- [ ] `cafe_noise.wav` (ガヤガヤした街頭やカフェの環境音)
  - **用途**: [demo_2_2_lfo_amp] 矩形波的なLFOで環境音をパルス状に切り刻み、環境音の中から無理やり注意を引く「通知システム」をシミュレーションする。

---

## 3. インタラクション用（手触りとシステム状況をデザインする）
ユーザーの操作やシステムの確率によって、デジタルデータが物理的な手触りや「不安定さ」を持つことを示します。

### 必要なファイル
- [ ] `speech_sentence.wav` (はっきりと聞き取れる日本語または英語の短いセリフ)
  - **用途**: [demo_3_1_interact_granular] 再生位置をスライダー（マウス）で操作。ゆっくり動かして声を引き伸ばしたり（タイムストレッチ）、逆再生させたりして「データに直接触れている感覚」を提示する。
- [ ] `orchestra_hit.wav` (ジャン！というオーケストラヒット音)
  - **用途**: [demo_3_1_interact_granular] グラニュラーで特定の瞬間をループさせ、強烈な電子音響（グリッチ・スタッター）を作り出す。
- [ ] `footsteps_walking.wav` または `clock_tick.wav` (一定テンポの足音や時計の音)
  - **用途**: [demo_3_2_interact_prob] 発音確率（Probability）を徐々に下げることで、確実だった足音や時計の音が歯抜けになり、システムの「バグ」やキャラクターの「疲労」を直感的に音響化する。
